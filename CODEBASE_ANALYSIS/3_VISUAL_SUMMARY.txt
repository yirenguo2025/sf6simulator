┃ CONNECTION VALIDATION LOGIC (in ComboEngine.addMove)                          ┃
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓

  addMove(move: MoveData)
  │
  ├─ First move? 
  │  └─→ ✅ Always succeeds (combo starts)
  │
  ├─ Find effective last move (skip Drive Rush)
  │
  ├─ Is it Drive Rush (66)?
  │  ├─ YES: Check if allowed from previous move's "Sp" property
  │  │       ├─→ ✅ dr_cancel connection
  │  │       └─→ ❌ Combo drops
  │  │
  │  └─ NO: Check cancel connection
  │     ├─ Cancel allowed?
  │     │  ├─ YES: Check if startup <= hitstun
  │     │  │       ├─→ ✅ cancel connection
  │     │  │       └─→ ❌ Not valid combo (but cancel exists)
  │     │  │
  │     │  └─ NO: Check link connection
  │     │     ├─ startup <= hitAdv?
  │     │     │  ├─→ ✅ link connection
  │     │     │  └─→ ❌ Combo drops
  │     │     
  │     └─ Neither: ❌ Combo drops, start fresh with this move
